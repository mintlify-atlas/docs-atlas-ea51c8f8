## Overview

Flash STU training is configured through a `config.json` file that controls model architecture, training hyperparameters, distributed training settings, and optimization flags.

## Model Parameters

These parameters define the architecture of the Flash STU model.

<ParamField path="model_type" type="string" default="FlashSTU">
  Model architecture type identifier
</ParamField>

<ParamField path="n_embd" type="integer" default="1536">
  Embedding dimension size. Controls the width of the model's hidden states.
</ParamField>

<ParamField path="n_heads" type="integer" default="8">
  Number of attention heads in multi-head attention layers.
</ParamField>

<ParamField path="n_layers" type="integer" default="26">
  Number of transformer layers in the model.
</ParamField>

<ParamField path="seq_len" type="integer" default="8192">
  Maximum sequence length the model can process.
</ParamField>

<ParamField path="window_size" type="integer" default="1024">
  Local attention window size for efficient processing.
</ParamField>

<ParamField path="vocab_size" type="integer" default="200064">
  Size of the token vocabulary. Should match the tokenizer's vocabulary size.
</ParamField>

<ParamField path="mlp_scale" type="integer" default="12">
  Scaling factor for MLP hidden dimension. MLP hidden size = n_embd * mlp_scale.
</ParamField>

<ParamField path="num_eigh" type="integer" default="24">
  Number of eigenvalues used in spectral filter computation for the STU layer.
</ParamField>

<ParamField path="bias" type="boolean" default="false">
  Whether to include bias terms in linear layers.
</ParamField>

<ParamField path="dropout" type="number" default="0.0">
  Dropout probability for regularization.
</ParamField>

<ParamField path="use_hankel_L" type="boolean" default="false">
  Whether to use Hankel-L matrix structure in spectral computations.
</ParamField>

<ParamField path="softcap" type="number" default="50.0">
  Softcap value for attention scores to prevent extreme values.
</ParamField>

## Training Parameters

These parameters control the training process and optimization.

<ParamField path="num_epochs" type="integer" default="1">
  Number of complete passes through the training dataset.
</ParamField>

<ParamField path="global_bsz" type="integer" default="524288">
  Global batch size in tokens across all GPUs. Must be divisible by `bsz * seq_len * world_size`.
</ParamField>

<ParamField path="bsz" type="integer" default="1">
  Per-GPU batch size (number of sequences per device).
</ParamField>

<ParamField path="warmup_steps" type="integer" default="19073">
  Number of warmup steps for learning rate schedule.
</ParamField>

<ParamField path="max_lr" type="number" default="3.0e-4">
  Maximum learning rate reached after warmup.
</ParamField>

<ParamField path="min_lr" type="number" default="3.0e-5">
  Minimum learning rate at the end of training (linear decay).
</ParamField>

<ParamField path="max_norm" type="number" default="1.0">
  Maximum gradient norm for gradient clipping.
</ParamField>

<ParamField path="eval_period" type="integer" default="25">
  Evaluate validation loss every N steps.
</ParamField>

<ParamField path="save_period" type="integer" default="19000">
  Save model checkpoint every N steps.
</ParamField>

<ParamField path="dilation" type="integer" default="1">
  Step dilation factor. Warmup, eval, and save periods are divided by this value.
</ParamField>

## Optimization Flags

These flags enable specific optimizations and features.

<ParamField path="use_flash_fft" type="boolean" default="true">
  Enable Flash FFT optimization for faster FFT operations in STU layers.
</ParamField>

<ParamField path="use_approx" type="boolean" default="true">
  Use approximation methods in spectral computations for improved performance.
</ParamField>

<ParamField path="use_attn" type="boolean" default="true">
  Enable attention mechanism in the model architecture.
</ParamField>

<ParamField path="torch_compile" type="boolean" default="false">
  Enable PyTorch 2.0 compilation for optimized execution. Can significantly improve throughput but increases compilation time.
</ParamField>

  "max_norm": 1.0,

  "fsdp_modules": ["STU", "Attention", "MLP"],

  "use_approx": true,
  "use_attn": true,
