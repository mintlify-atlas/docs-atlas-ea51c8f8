## Overview

The `Attention` module implements multi-head self-attention using Flash Attention 2 for efficient computation. It includes sliding window attention, ALiBi (Attention with Linear Biases) positional encoding, and optional attention softcapping.

**Source:** `flash_stu/modules/attention.py`

### Linear Projections

<ResponseField name="c_attn" type="nn.Linear">

<ResponseField name="n_heads" type="int">

<ResponseField name="window_size" type="int">

#### Processing Steps

1. **QKV Projection:** Projects input through `c_attn` and splits into Q, K, V
2. **Reshape:** Reshapes Q, K, V to `(bsz, seq_len, n_heads, head_dim)`
3. **Flash Attention:** Applies efficient attention computation with:
   - Causal masking (autoregressive)
   - Sliding window `(window_size, 0)` - attends to left context only
   - ALiBi slopes for positional encoding
   - Softcap for stability
   - Dropout during training
4. **Output Projection:** Reshapes and projects through `c_proj`
5. **Residual Dropout:** Applies dropout to output

## Flash Attention Integration

The module uses the Triton-based Flash Attention 2 implementation:

```python
from flash_attn import flash_attn_func as fa2

y = fa2(
    q, k, v,
    dropout_p=self.dropout if self.training else 0.0,
    causal=True,
    window_size=(self.window_size, 0),
    alibi_slopes=self.alibi_slopes,
    softcap=self.softcap,
)
```

**Parameters:**
- `dropout_p`: Attention dropout (only during training)
- `causal=True`: Autoregressive masking
- `window_size=(window_size, 0)`: Sliding window with left-only context
- `alibi_slopes`: Per-head positional bias
- `softcap`: Logit capping for stability

**Reference:** [Flash Attention 2 paper](https://arxiv.org/pdf/2307.08691)

## Sliding Window Mechanism

The sliding window restricts attention to local context:

```python
window_size=(self.window_size, 0)
```

- **Left context:** `window_size` tokens (e.g., 1024)
- **Right context:** 0 tokens (causal)
- **Effect:** Each token attends to at most `window_size` previous tokens
- **Benefit:** O(n × window_size) complexity instead of O(n²)

## Softcap Parameter

Softcapping prevents attention logits from growing too large:

```python
softcap=self.softcap  # e.g., 50.0
```

**Implementation:** `logits = softcap * tanh(logits / softcap)`

**Benefits:**
- Prevents numerical instability
- Smooths attention distribution
- Improves training stability

**Reference:** [Gemini paper](https://arxiv.org/pdf/2408.00118)

## Usage Example

```python
import torch
from flash_stu.config import FlashSTUConfig
from flash_stu.modules.attention import Attention

# Create configuration
config = FlashSTUConfig(
    n_embd=1536,
    n_heads=8,
    bsz=4,
    window_size=1024,
    softcap=50.0,
    dropout=0.0,
    bias=False,
    torch_dtype=torch.bfloat16
)

# Initialize attention module
attn = Attention(config).cuda()

# Forward pass
batch_size = 4
seq_len = 2048
x = torch.randn(batch_size, seq_len, config.n_embd,
                dtype=torch.bfloat16, device='cuda')
output = attn(x)  # Shape: (4, 2048, 1536)

# Check ALiBi slopes
print(f"ALiBi slopes: {attn.alibi_slopes}")  # Shape: (8,)
```

## ALiBi Positional Encoding

ALiBi (Attention with Linear Biases) adds positional information through learned slopes:

- **No absolute position embeddings** needed
- **Relative position bias:** Each head gets a unique slope
- **Extrapolation:** Better generalization to longer sequences
- **Interpolation factor:** 0.25 for improved performance

**Reference:** [ALiBi paper](https://arxiv.org/pdf/2108.12409)

## Performance Characteristics

- **Memory efficient:** O(n) memory via Flash Attention 2
- **Compute efficient:** Optimized CUDA kernels
- **Sliding window:** Reduces complexity for long sequences
- **Causal masking:** Built into kernel (no explicit mask needed)
