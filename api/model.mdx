## FlashSTU

The main model class implementing the Flash STU architecture. Inherits from `transformers.PreTrainedModel`.

### Constructor

```python
FlashSTU(config: FlashSTUConfig, phi: torch.Tensor) -> None
```

Initializes the FlashSTU model with the specified configuration and eigenvalue matrix.

<ParamField path="config" type="FlashSTUConfig" required>
  Configuration object containing all model hyperparameters. See [FlashSTUConfig](/api/config) for details.
</ParamField>

<ParamField path="phi" type="torch.Tensor" required>
  Eigenvalue matrix used for STU layer computations. This matrix contains precomputed eigenvalues for the structured transform.
</ParamField>

**Implementation Details:**
- Automatically sets `config_class = FlashSTUConfig`
- Creates token embeddings and dropout layer
- Builds alternating STU and Attention layers (even layers use STU, odd layers use Attention if `use_attn=True`)
- Applies RMSNorm (Triton-optimized if available, otherwise PyTorch)
- Initializes language model head with weight tying to token embeddings
- Applies custom weight initialization via `_init_weights`
- Prints total parameter count on initialization

**Example:**
```python
import torch
from flash_stu import FlashSTU, FlashSTUConfig

# Create configuration
config = FlashSTUConfig(
    n_embd=1536,
    n_layers=26,
    seq_len=8192,
    vocab_size=200064
)

# Precompute eigenvalue matrix (phi)
phi = torch.randn(config.num_eigh, config.seq_len)

# Initialize model
model = FlashSTU(config, phi)
```

### Methods

#### forward

```python
forward(x: torch.Tensor) -> torch.Tensor
```

Performs forward pass through the model.

<ParamField path="x" type="torch.Tensor" required>
  Input tensor of token indices with shape `(batch_size, sequence_length)`.
</ParamField>

<ResponseField name="y_hat" type="torch.Tensor">
  Output logits with shape `(batch_size, sequence_length, vocab_size)`.
</ResponseField>

**Forward Pass Steps:**
1. Token embedding lookup
2. Dropout application
3. Sequential layer processing (STU/Attention layers)
4. RMS normalization
5. Language model head projection

**Example:**
```python
# Prepare input tokens
input_ids = torch.randint(0, config.vocab_size, (2, 512))

# Forward pass
logits = model.forward(input_ids)
print(logits.shape)  # torch.Size([2, 512, 200064])
```

#### _get_num_params

```python
_get_num_params() -> int
```

Calculates the total number of trainable parameters in the model.

<ResponseField name="n_params" type="int">
  Total number of parameters, excluding:
  - Positional embeddings (if present)
  - Token embedding weights (if not tied with LM head)
</ResponseField>

**Note:** This method accounts for weight tying between token embeddings and the language model head to avoid double-counting parameters.

**Example:**
```python
num_params = model._get_num_params()
print(f"Model has {num_params / 1e6:.2f}M parameters")
```

#### _init_weights

```python
_init_weights(module: nn.Module) -> None
```

Initializes weights for different module types with custom strategies.

<ParamField path="module" type="nn.Module" required>
  PyTorch module to initialize. Called automatically during model construction via `self.apply()`.
</ParamField>

**Initialization Strategy:**

- **Linear layers**: Normal distribution with `mean=0.0`, `std=(n_embd)^(-0.5)`. Layers with `SCALE_INIT` attribute use scaled std: `std *= (2 * n_layers)^(-0.5)`. Biases initialized to zeros.

- **Embedding layers**: Normal distribution with `mean=0.0`, `std=(n_embd)^(-0.5)`.

- **STU modules**: Xavier normal initialization for:
  - `M_inputs` and `M_filters` (if `use_approx=True`)
  - `M_phi_plus` and `M_phi_minus` (if `use_approx=False`)

- **Attention modules**: Xavier normal initialization for `c_attn` and `c_proj` weights. Biases initialized to zeros.

**Example:**
```python
# Custom module initialization
custom_layer = nn.Linear(1536, 1536)
model._init_weights(custom_layer)
```

## Class Attributes

<ParamField path="config_class" type="Type[FlashSTUConfig]">
  Reference to the configuration class used by this model.
</ParamField>

<ParamField path="n_layers" type="int">
  Number of transformer layers in the model.
</ParamField>

<ParamField path="n" type="int">
  Nearest power of two for `seq_len * 2 - 1`, used for FFT operations.
</ParamField>

<ParamField path="phi" type="torch.Tensor">
  Eigenvalue matrix for STU computations.
</ParamField>

<ParamField path="use_approx" type="bool">
  Whether to use approximate STU computation.
</ParamField>

<ParamField path="use_hankel_L" type="bool">
  Whether to use Hankel-L structured matrices.
</ParamField>
