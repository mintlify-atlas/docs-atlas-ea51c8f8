This guide will help you get up and running with Flash STU quickly. We'll walk through a basic example of creating and using a Flash STU model.

## Basic Usage

Here's a minimal example showing how to import Flash STU, configure it, and run a forward pass:

```python
from flash_stu import FlashSTU, FlashSTUConfig, get_spectral_filters
import torch

device = torch.device('cuda')  # Flash STU requires CUDA

# Step 1: Create configuration
config = FlashSTUConfig(
    bsz=1,              # Batch size
    n_embd=1536,        # Embedding dimension
    n_heads=8,          # Number of attention heads
    n_layers=26,        # Number of layers
    seq_len=8192,       # Sequence length
    window_size=1024,   # Attention window size
    vocab_size=200064,  # Vocabulary size
    mlp_scale=12,       # MLP hidden dimension scale
    bias=False,         # Use bias in linear layers
    dropout=0.0,        # Dropout probability
    num_eigh=24,        # Number of eigenvalues for spectral filters
    use_hankel_L=False, # Use Hankel L matrix
    use_flash_fft=True, # Use Flash FFT for convolutions
    use_approx=True,    # Use approximation method
    use_attn=True,      # Use attention layers
    softcap=50.0,       # Softmax cap value
    torch_dtype=torch.bfloat16,  # Data type
)

# Step 2: Get spectral filters
phi = get_spectral_filters(
    config.seq_len,
    config.num_eigh,
    config.use_hankel_L,
    device,
    config.torch_dtype
)

# Step 3: Initialize model
model = FlashSTU(config, phi)
model = model.to(device)

# Step 4: Run forward pass
# Input shape: (batch_size, sequence_length)
x = torch.randint(0, config.vocab_size, (config.bsz, config.seq_len), device=device)
y = model(x)

print(f"Input shape: {x.shape}")
print(f"Output shape: {y.shape}")  # (batch_size, sequence_length, vocab_size)
```

## Understanding the Components

<Steps>
  <Step title="Configuration">
    The `FlashSTUConfig` class contains all hyperparameters for the model. Key parameters include:

    - `n_embd`: Embedding dimension (model width)
    - `n_layers`: Number of transformer-like layers
    - `seq_len`: Maximum sequence length
    - `num_eigh`: Number of eigenvalues for spectral filters (controls spectral decomposition rank)
    - `use_flash_fft`: Enable Flash FFT for faster convolutions
  </Step>

  <Step title="Spectral Filters">
    Spectral filters (`phi`) are computed based on the Hankel matrix eigendecomposition:

    ```python
    phi = get_spectral_filters(
        seq_len,       # Sequence length
        num_eigh,      # Number of top eigenvalues to use
        use_hankel_L,  # Which Hankel matrix variant to use
        device,        # CUDA device
        torch_dtype    # Data type
    )
    ```

    These filters are essential for the spectral transform and should match your sequence length.
  </Step>

  <Step title="Model Initialization">
    Create the Flash STU model with your config and spectral filters:

    ```python
    model = FlashSTU(config, phi)
    ```

    The model alternates between STU layers and attention layers (if `use_attn=True`).
  </Step>

  <Step title="Forward Pass">
    Run inference by passing token IDs to the model:

    ```python
    # x: (batch_size, seq_len) - token indices
    y = model(x)
    # y: (batch_size, seq_len, vocab_size) - logits
    ```

    The output contains logits for next-token prediction.
  </Step>
</Steps>

## Training Example

Here's a more complete example showing how to set up training:

```python
from flash_stu import FlashSTU, FlashSTUConfig, get_spectral_filters
import torch
from torch.nn import CrossEntropyLoss
from torch.optim import AdamW

device = torch.device('cuda')

# Configuration
config = FlashSTUConfig(
    bsz=8,
    n_embd=1536,
    n_heads=8,
    n_layers=26,
    seq_len=8192,
    window_size=1024,
    vocab_size=200064,
    mlp_scale=12,
    num_eigh=24,
    use_hankel_L=False,
    use_flash_fft=True,
    use_approx=True,
    use_attn=True,
    torch_dtype=torch.bfloat16,
)

# Get spectral filters
phi = get_spectral_filters(
    config.seq_len,
    config.num_eigh,
    config.use_hankel_L,
    device,
    config.torch_dtype
)

# Initialize model
model = FlashSTU(config, phi).to(device)

# Setup optimizer and loss
optimizer = AdamW(model.parameters(), lr=3e-4)
loss_fn = CrossEntropyLoss()

# Training loop
model.train()
for step in range(100):
    # Generate dummy data (replace with your dataloader)
    inputs = torch.randint(0, config.vocab_size, (config.bsz, config.seq_len), device=device)
    targets = torch.randint(0, config.vocab_size, (config.bsz, config.seq_len), device=device)
    
    # Forward pass
    logits = model(inputs)
    
    # Compute loss
    loss = loss_fn(logits.flatten(0, 1), targets.flatten(0, 1))
    
    # Backward pass
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if step % 10 == 0:
        print(f"Step {step}, Loss: {loss.item():.4f}")
```

<Note>
  For production training, see the complete example in `training/example.py` which includes:
  - Distributed training with DDP/FSDP
  - Gradient accumulation
  - Mixed precision training
  - Checkpointing
  - Learning rate scheduling
</Note>

## Key Configuration Options

<AccordionGroup>
  <Accordion title="Model Architecture">
    - `n_embd`: Embedding dimension (default: 1536)
    - `n_heads`: Number of attention heads (default: 8)
    - `n_layers`: Number of layers (default: 26)
    - `seq_len`: Maximum sequence length (default: 8192)
    - `window_size`: Sliding window size for attention (default: 1024)
  </Accordion>

  <Accordion title="Spectral Transform">
    - `num_eigh`: Number of eigenvalues for spectral filters (default: 24)
    - `use_hankel_L`: Use Hankel L matrix variant (default: False)
    - `use_approx`: Use approximation method for efficiency (default: True)
  </Accordion>

  <Accordion title="Performance Optimizations">
    - `use_flash_fft`: Enable Flash FFT convolutions (default: True)
    - `use_attn`: Include attention layers (default: True)
    - `torch_dtype`: Data type for model weights (default: torch.bfloat16)
  </Accordion>

  <Accordion title="Training">
    - `bsz`: Batch size (default: 1)
    - `dropout`: Dropout probability (default: 0.0)
    - `bias`: Use bias in linear layers (default: False)
    - `mlp_scale`: MLP hidden dimension multiplier (default: 12)
  </Accordion>
</AccordionGroup>
