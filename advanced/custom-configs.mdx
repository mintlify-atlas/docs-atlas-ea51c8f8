## Overview

Flash STU offers extensive configuration options to customize model architecture for your specific use case. This guide explains each parameter and provides best practices for model design.

## Core Architecture Parameters

### Embedding Dimension (`n_embd`)

The primary dimension of the model's hidden representations.

```python
config = FlashSTUConfig(
    n_embd=1536,  # Default value
)
```

**Common Values:**
- Small models: 512, 768
- Medium models: 1024, 1536
- Large models: 2048, 4096

**Considerations:**
- Larger `n_embd` increases model capacity and parameter count quadratically
- Must be divisible by `n_heads` for attention layers
- Affects memory usage significantly (most parameters scale with `n_embd²`)

<Tip>
Start with `n_embd=1024` for models under 500M parameters, and `n_embd=1536` or higher for larger models. Ensure it's divisible by your chosen `n_heads`.
</Tip>

See `flash_stu/config.py:12` and `flash_stu/config.py:38`.

### Number of Attention Heads (`n_heads`)

Controls multi-head attention parallelism in attention layers.

```python
config = FlashSTUConfig(
    n_heads=8,  # Default value
)
```

**Common Values:**
- 8 heads for models with `n_embd=1024-1536`
- 16 heads for models with `n_embd=2048-3072`
- 32 heads for models with `n_embd=4096+`

**Constraints:**
- `n_embd` must be divisible by `n_heads` (see `flash_stu/modules/attention.py:20`)
- Head dimension: `d_head = n_embd / n_heads`
- Typical head dimensions: 64, 128, or 256

<Warning>
If `n_embd % n_heads != 0`, the model will raise an assertion error during initialization. Always verify this constraint.
</Warning>

**ALiBi Slopes:**
- Flash STU uses ALiBi (Attention with Linear Biases) for position encoding
- Slopes are automatically generated for any `n_heads` value
- Non-power-of-2 head counts are supported with interpolated slopes

See `flash_stu/modules/attention.py:42-57`.

### Number of Layers (`n_layers`)

Total number of transformer layers (STU + attention).

```python
config = FlashSTUConfig(
    n_layers=26,  # Default value
)
```

**Guidelines:**
- Small models (< 500M params): 12-18 layers
- Medium models (500M-1B params): 20-28 layers
- Large models (1B+ params): 32-48+ layers

**Layer Arrangement:**
By default, Flash STU alternates between STU and attention layers:
- Even indices (0, 2, 4, ...): STU layers
- Odd indices (1, 3, 5, ...): Attention layers (if `use_attn=True`) or STU layers

See `flash_stu/model.py:43-52` for the layer construction logic.

<Tip>
Deeper models generally perform better but require more memory and compute. Use gradient checkpointing for very deep models (40+ layers).
</Tip>

## Spectral Transform Parameters

### Number of Eigenvectors (`num_eigh`)

Controls the rank of the spectral filter approximation.

```python
config = FlashSTUConfig(
    num_eigh=24,  # Default value
)
```

**Impact:**
- Higher `num_eigh` = more expressive spectral filters
- Directly affects STU layer parameter count
- With `use_approx=True`: `K × d_in` parameters per filter
- With `use_approx=False`: `K × d_in × d_out` parameters

**Recommended Values:**
- 16-24 for most applications
- 32-48 for tasks requiring fine-grained spectral modeling
- 8-16 for memory-constrained settings

<Tip>
`num_eigh=24` provides a good balance between expressiveness and efficiency. Increase to 32-48 only if you have evidence that spectral capacity is limiting performance.
</Tip>

The eigenvectors are computed from the Hankel matrix via eigendecomposition:
```python
phi = get_spectral_filters(
    config.seq_len,
    config.num_eigh,
    config.use_hankel_L,
    device,
    config.torch_dtype
)
```

See `flash_stu/utils/stu_utils.py:25-37` and `flash_stu/config.py:21`.

### Window Size for Attention (`window_size`)

Defines the sliding window size for local attention.

```python
config = FlashSTUConfig(
    window_size=1024,  # Default value
)
```

**Behavior:**
- Each token attends to the previous `window_size` tokens (causal)
- Reduces attention complexity from O(n²) to O(n × window_size)
- Passed to Flash Attention 2 as `window_size=(window_size, 0)`

**Recommended Values:**
- 512 for short-context tasks (< 4K tokens)
- 1024 for medium-context tasks (4K-8K tokens)
- 2048+ for long-context tasks (8K+ tokens)

<Warning>
Larger window sizes increase memory usage quadratically within the attention layer. Monitor GPU memory when increasing `window_size`.
</Warning>

See `flash_stu/modules/attention.py:35` and `flash_stu/modules/attention.py:74`.

## Vocabulary and Sequence Parameters

### Vocabulary Size (`vocab_size`)

Number of unique tokens in your tokenizer.

```python
config = FlashSTUConfig(
    vocab_size=200064,  # Default value
)
```

**Guidelines:**
- Must match your tokenizer's vocabulary size
- Common values: 50257 (GPT-2), 32000 (LLaMA), 200064 (custom)
- Affects embedding and LM head parameters: `vocab_size × n_embd × 2`

<Tip>
Round vocabulary size to multiples of 64 or 128 for better GPU memory alignment. For example, use 32000 instead of 31987.
</Tip>

Note: Flash STU uses weight tying (`tok_emb.weight = lm_head.weight`), so embedding parameters are shared (see `flash_stu/model.py:64`).

### Sequence Length (`seq_len`)

Maximum sequence length the model can process.

```python
config = FlashSTUConfig(
    seq_len=8192,  # Default value
)
```

**Important Notes:**
- Flash FFT pads to nearest power of 2: `n = nearest_power_of_two(seq_len * 2 - 1)`
- Prefer power-of-2 values: 1024, 2048, 4096, 8192, 16384
- Longer sequences require more memory and compute

See `flash_stu/model.py:31` for the padding calculation.

## Layer Arrangement Patterns

### STU and Attention Layer Splits

The default implementation uses a 50/50 alternating pattern (controlled by `use_attn`):

```python
# Default pattern in model.py:43-52
for layer_idx in range(self.n_layers):
    if layer_idx % 2 == 0:
        self.layers.append(STULayer(config, self.phi, self.n))
    else:
        self.layers.append(
            AttentionLayer(config) if config.use_attn 
            else STULayer(config, self.phi, self.n)
        )
```

**Pattern Examples:**

1. **All STU** (`use_attn=False`):
   ```python
   config = FlashSTUConfig(use_attn=False)
   # Layers: [STU, STU, STU, STU, ...]
   ```

2. **50/50 Alternating** (`use_attn=True`, default):
   ```python
   config = FlashSTUConfig(use_attn=True)
   # Layers: [STU, Attn, STU, Attn, ...]
   ```

3. **Custom Patterns**:
   For more complex arrangements (e.g., 75% STU, 25% attention), see the [Spectral State Space Models paper](https://arxiv.org/pdf/2406.07887) referenced in `flash_stu/model.py:44`.

**Customization Example:**

```python
# Modify model.py for custom pattern
# Example: Attention every 4 layers
for layer_idx in range(self.n_layers):
    if layer_idx % 4 == 3:  # Layers 3, 7, 11, 15...
        self.layers.append(AttentionLayer(config))
    else:
        self.layers.append(STULayer(config, self.phi, self.n))
```

<Tip>
The 50/50 alternating pattern works well for most language modeling tasks. Experiment with different ratios only if you have specific architectural requirements.
</Tip>

## Advanced Parameters

### Softcap (`softcap`)

Applies attention logit capping for improved training stability.

```python
config = FlashSTUConfig(
    softcap=50.0,  # Default value
)
```

**Mechanism:**
- Caps attention logits using `tanh`: `logits = softcap × tanh(logits / softcap)`
- Prevents extremely large attention scores
- Technique from [Gemini technical report](https://arxiv.org/pdf/2408.00118)

**Recommended Values:**
- 30.0 - 50.0 for most models
- 20.0 for smaller models or aggressive capping
- Higher values (100.0+) for minimal capping effect

See `flash_stu/modules/attention.py:36` and `flash_stu/modules/attention.py:76`.

### MLP Scale (`mlp_scale`)

Ratio of MLP hidden size to embedding dimension.

```python
config = FlashSTUConfig(
    mlp_scale=12,  # Default: intermediate_size = n_embd * 12
)
```

**Calculation:**
```python
self.intermediate_size = n_embd * mlp_scale
```

**Guidelines:**
- Standard Transformers use 4x (e.g., GPT-2, BERT)
- Flash STU defaults to 12x for increased MLP capacity
- Larger ratios increase parameter count significantly

See `flash_stu/config.py:18` and `flash_stu/config.py:39`.

### Bias Terms (`bias`)

Enables/disables bias terms in linear layers.

```python
config = FlashSTUConfig(
    bias=False,  # Default: no bias terms
)
```

<Tip>
Modern LLMs typically disable bias (`bias=False`) to reduce parameters without significant performance loss. Enable only if you have specific architectural reasons.
</Tip>

### Dropout (`dropout`)

Dropout probability for regularization.

```python
config = FlashSTUConfig(
    dropout=0.0,  # Default: no dropout
)
```

**Usage:**
- Applied in: token embeddings, attention, and residual connections
- Typically 0.0 for large-scale pretraining
- 0.1-0.2 for fine-tuning or smaller datasets

## Parameter Count Estimation

Estimate total parameters with this formula:

```
Embedding: vocab_size × n_embd
STU Layers (with approx): n_stu_layers × (n_embd² + num_eigh × n_embd)
Attention Layers: n_attn_layers × (4 × n_embd²)
MLP Layers: n_layers × (2 × n_embd × intermediate_size)
Normalization: n_layers × n_embd
```

<Tip>
Use the model's built-in parameter counting:
```python
model = FlashSTU(config, phi)
# Prints: "Model Parameter Count: XXX.XXM"
```
See `flash_stu/model.py:68` and `flash_stu/model.py:82-88`.
</Tip>
